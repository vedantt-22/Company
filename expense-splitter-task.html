Task: Expense Splitter (Easy → Medium, Time-Consuming)

Rules: use arrays only (no objects), and every add/edit/delete must clear + re-render the UI.
Student Task Description

Build a small “expense splitter” web page using only vanilla HTML + JavaScript. Users will enter a person’s name and an expense amount, and your app will list all entries and show summary statistics.

This task is intentionally written to practice: input validation, array operations (push, splice), indexing, and updating the DOM by re-rendering.

  • Data rule: store all entries using exactly two parallel arrays: people[] and amounts[].
  • No objects: do not store a row as { name, amount } or any other object/array-of-objects structure.
  • Render rule: whenever the arrays change (add/edit/delete/clear), you must rebuild the UI from scratch by calling render().
  • Browser APIs: for edit/delete confirmations and inputs, use confirm() and prompt() as specified.
  • Deliverable: a single working HTML file where all features in the Requirements section work correctly.

Tip: keep your logic simple—update the arrays first, then call render(). Avoid “partial updates” to the table.

Requirements
  1. Store data in two parallel arrays:
    • people[] (names)
    • amounts[] (numbers)
  2. Add expense:
    • Inputs: person name, amount
    • Validation: name length ≥ 2, amount is a number and > 0
    • On success: push into arrays and re-render
  3. Edit expense:
    • Click Edit button on a row
    • Ask confirm() first
    • Use prompt() to get new name + new amount
    • Update arrays at the same index and re-render
  4. Delete expense:
    • Double click a row OR click Delete button (your choice, but keep it simple)
    • Ask confirm() then remove by index using splice
    • Re-render
  5. Stats panel (computed each render):
    • Total expenses count
    • Total amount (sum)
    • Split per person = total / unique people count
  6. Must show amounts with 2 decimals using toFixed(2).
